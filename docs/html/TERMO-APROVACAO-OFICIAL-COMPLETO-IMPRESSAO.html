OBJETIVO PRINCIPAL

Desenvolver um sistema web completo que unifique a comunicação e serviços da Paróquia Nossa Senhora Aparecida com suas 7 comunidades em Itapetininga/SP, fornecendo:

  • Presença digital moderna para a paróquia
  • Centralização de informações paroquiais
  • Recursos espirituais online acessíveis 24/7
  • Sistema de gestão de missas e eventos
  • Comunicação eficiente com os fiéis
FUNCIONALIDADES IMPLEMENTADAS
1. Sistema Frontend Angular 20
  • Página Inicial Responsiva - Design moderno com vídeo de fundo
  • Bíblia Online Completa - 66 livros, busca avançada, navegação por capítulos
  • Horários de Missa - Todas as 7 comunidades organizadas por dia
  • liturgia Diária - Leituras, evangelho, salmos do dia
  • Santo do Dia - Biografia, imagens e informações litúrgicas
  • Sistema de Notícias - Eventos e campanhas paroquiais
  • Gestão de Grupos - Pastorais e movimentos
2. Sistema Backend Spring Boot 3.5.5
  • API RESTful Completa - Endpoints para todos os recursos
  • Banco de Dados H2 - Persistência de dados dos santos
  • Sistema de Santos - Base de dados litúrgica completa
  • Web Scraping Inteligente - Coleta automática de informações
  • Sistema de Importação - Upload manual de dados
  • Gestão de Imagens Legais - Apenas fontes de domínio público
3. Integrações de APIs Públicas
  • Bible API - bible-api.com (Tradução Almeida Portuguesa)
  • liturgia API - liturgia.up.railway.app (liturgia diária oficial)
  • Wikimedia Commons - Imagens de santos (domínio público)
  • Vatican Official - Fontes confiáveis da Igreja
ESTRUTURA DO BANCO DE DADOS
Tabela: santos
Campo Tipo Descrição
id BIGINT Identificador único
nome VARCHAR(200) Nome completo do santo
titulo VARCHAR(300) Título honorífico
biografia TEXT Vida e obra do santo
dia INTEGER Dia da festa litúrgica
mes INTEGER Mês da festa litúrgica
imagem_url VARCHAR(500) URL da imagem (domínio público)
fonte VARCHAR(100) Fonte da informação
Configuração do Banco
  • Tipo: H2 Database (file-based)
  • Localização: ./dados/santos_db
  • Persistência: Dados mantidos entre execuções
  • Console Web: Disponível para administração
  • Backup: Automático via Spring Boot
