Entidade Principal - Santo.java
@Entity @Table(name = "santos") public class Santo { @Id @GeneratedValue private Long id; @NotBlank @Column(length = 200) private String nome; @Column(length = 300) private String titulo; @Column(columnDefinition = "TEXT") private String biografia; @NotNull private Integer dia; @NotNull private Integer mes; @Column(name = "imagem_url") private String imagemUrl; @Column(length = 100) private String fonte; @CreationTimestamp private LocalDateTime createdAt; @UpdateTimestamp private LocalDateTime updatedAt; }
ENDPOINTS DA API REST
Santos - Consulta
GET /api/santos/hoje
Retorna: Santo do dia atual
GET /api/santos/data/{dia}/{mes}
Parâmetros: dia (1-31), mes (1-12)
Retorna: Lista de santos da data específica
GET /api/santos/mes/{mes}
Parâmetros: mes (1-12)
Retorna: Todos os santos do mês
GET /api/santos/buscar?nome={nome}
Parâmetros: nome (string)
Retorna: Santos com nome similar
Santos - Gestão (CRUD)
POST /api/santos
Body: JSON Santo
Retorna: Santo criado com ID
PUT /api/santos/{id}
Parâmetros: id (Long)
Body: JSON Santo
Retorna: Santo atualizado
DELETE /api/santos/{id}
Parâmetros: id (Long)
Retorna: Status da operação
Importação de Dados
POST /api/santos/importar/arquivo
Body: MultipartFile (TXT)
Retorna: Relatório de importação
GET /api/santos/importar/exemplo
Retorna: Exemplo de formato de dados
SISTEMA ADMINISTRATIVO
Funcionalidades da Área Admin
  • Autenticação Segura: Login com usuário e senha criptografada
  • Gestão de Usuários: Criar, editar e gerenciar usuários administrativos
  • Gestão de Conteúdo: Adicionar, editar e excluir santos
  • Dashboard Estatísticas: Visão geral do sistema
  • Controle de Permissões: Diferentes níveis de acesso (Admin, Editor, Visualizador)
Entidade Usuário.java
@Entity @Table(name = "Usuários") public class Usuário { @Id @GeneratedValue(strategy = GenerationType.IDENTITY) private Long id; @Column(nullable = false, unique = true) private String username; @Email @Column(nullable = false, unique = true) private String email; @Column(nullable = false) private String senha; // Hash SHA-256 @Column(name = "nome_completo") private String nomeCompleto; @Enumerated(EnumType.STRING) private Role role; // ADMIN, EDITOR, VISUALIZADOR private Boolean ativo = true; private LocalDateTime criadoEm; private LocalDateTime ultimoLogin; }
Níveis de Permissão
Role Permissões Uso Recomendado
ADMIN Acesso total: usuários, conteúdo, configurações Padre pároco, coordenador técnico
EDITOR Adicionar/editar conteúdo (santos, notícias) Membros da equipe pastoral
VISUALIZADOR Apenas visualizar relatórios e estatísticas Consulta e acompanhamento
APIs Administrativas
# Autenticação POST /api/admin/login Body: {"login": "admin", "senha": "admin123"} # Dashboard com estatísticas GET /api/admin/dashboard Response: { "Usuários": {"total": 3, "ativos": 2}, "santos": {"total": 365, "comImagem": 200} } # Gestão de usuários GET /api/admin/Usuários # Listar usuários POST /api/admin/Usuários # Criar usuário PUT /api/admin/Usuários/{id}/status # Ativar/desativar PUT /api/admin/Usuários/{id}/role # Alterar permissão # Gestão de conteúdo (santos) GET /api/santos # Listar santos POST /api/santos # Criar santo PUT /api/santos/{id} # Editar santo DELETE /api/santos/{id} # Excluir santo
Segurança Implementada
  • Hash de Senhas: SHA-256 para proteger credenciais
  • Validação de Dados: Verificação de entrada em todos os endpoints
  • Controle de Sessão: Registro de último login
  • Auditoria: Log de criação e modificação de registros
  • Fonte de Imagens: Validação de URLs apenas de fontes confiáveis

DEPLOY E CONFIGURAÇÃO
Requisitos do Sistema
Java 21 LTS
Maven 3.9+
RAM Mínimo 512MB
Espaço em Disco 1GB
Porta 8080 disponível
Configuração de Desenvolvimento
1. Clonar repositório 2. ./mvnw clean install 3. ./mvnw spring-boot:run 4. Acessar http://localhost:8080
Variáveis de Ambiente
  • SPRING_PROFILES_ACTIVE=prod
  • DATABASE_PATH=./dados/santos_db
  • CORS_ORIGINS=https://paróquia.com.br
  • LOG_LEVEL=INFO
SEGURANÇA E VALIDAÇÃO
Configuração CORS
  • Permite origins: http://localhost:4200
  • Métodos: GET, POST, PUT, DELETE, OPTIONS
  • Headers: Content-Type, Authorization
  • Credentials: false
Validação de Dados
  • @NotBlank: nome obrigatório
  • @NotNull: dia e mês obrigatórios
  • @Valid: validação automática de entidades
  • @Size: limitação de tamanho de strings
  • Validação customizada para datas
Tratamento de Erros
  • GlobalExceptionHandler para exceções globais
  • ResponseEntity com status HTTP apropriados
  • Logs detalhados para debugging
  • Mensagens de erro amigáveis
MONITORAMENTO E LOGS
Logs Configurados
